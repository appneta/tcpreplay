// Shared tcpreplay option documentation, included by both tcpreplay.1.adoc and
// tcpreplay-edit.1.adoc.  Does not include --version/--less-help/--help, which
// each man page lists at the end of its OPTIONS section.

*-d, --dbug*=_number_::
  Enable debugging output. Takes an integer in the range 0 through 5 (default
  0). Only available if the program was configured with *--enable-debug*;
  higher numbers increase verbosity.

*-q, --quiet*::
  Quiet mode; print nothing except the statistics at the end of the run.

*-T, --timer*=_mode_::
  Select the packet timing method: _nano_ (nanosleep()), _select_ (select()),
  _ioport_ (i386 IO port 0x80) or _gtod_ (gettimeofday() loop, the default).

*--maxsleep*=_number_::
  Sleep for no more than X milliseconds between packets, preventing long delays
  without affecting the majority of packets. Default is disabled.

*-v, --verbose*::
  Print decoded packets via tcpdump to STDOUT. Only available if the program
  was configured with *--enable-verbose*.

*-A, --decode*=_string_::
  Arguments passed to the tcpdump decoder when *--verbose* is used. By default
  -n and -l are used. Be sure to quote the arguments so they are not interpreted
  by tcpreplay. Requires *--verbose*.

*-K, --preload-pcap*::
  Preload the pcap(s) into RAM before sending to improve replay performance (at
  the cost of a startup delay). Also suppresses per-iteration flow statistics
  collection, which can significantly reduce memory usage.

*-c, --cachefile*=_string_::
  Split traffic across two interfaces using a tcpprep(1) cache file. Cannot be
  combined with *--dualfile*; requires *--intf2*.

*-2, --dualfile*::
  Replay two files at a time from a network tap (one per direction), inter-mixed
  using the timestamps in each. Cannot be combined with *--cachefile*; requires
  *--intf2*.

*-i, --intf1*=_string_::
  Client to server / RX / primary traffic output interface. Required (unless
  *--write* is used).

*-I, --intf2*=_string_::
  Server to client / TX / secondary traffic output interface. Generally only
  useful with *--cachefile* or *--dualfile*.

*-w, --write*=_string_::
  Write traffic to a pcap file instead of an interface. Cannot be combined with
  *--intf2*.

*--include*=_LIST_::
  Send only the selected packet numbers, e.g. _--include=1-5,9,15,72-_. Cannot
  be combined with *--exclude*.

*--exclude*=_LIST_::
  Send all but the selected packet numbers. Cannot be combined with *--include*.

*--listnics*::
  List available network interfaces and exit. Only available if configured with
  pcap findalldevs support.

*-l, --loop*=_number_::
  Loop through the capture file X times (0 = loop forever). Default is 1.

*--loopdelay-ms*=_number_::
  Delay between loops in milliseconds. Requires *--loop*; cannot be combined
  with *--loopdelay-ns*.

*--loopdelay-ns*=_number_::
  Delay between loops in nanoseconds (requires *nano* timing mode). Requires
  *--loop*; cannot be combined with *--loopdelay-ms*.

*--pktlen*::
  Override the snaplen and use the actual packet length stored in the header.
  Use with caution.

*-L, --limit*=_number_::
  Limit the number of packets to send. By default all packets are sent.

*--duration*=_number_::
  Limit the number of seconds to send. By default all packets are sent.

*-x, --multiplier*=_string_::
  Modify the replay speed to a given multiple of the captured rate. Cannot be
  combined with *--pps*, *--mbps*, *--oneatatime* or *--topspeed*.
+
....
--multiplier=2.0    # replay at twice the captured speed
--multiplier=0.7    # replay at 70% of the captured speed
....

*-p, --pps*=_string_::
  Replay packets at a given packets/sec rate. Cannot be combined with
  *--multiplier*, *--mbps*, *--oneatatime* or *--topspeed*.
+
....
--pps=200     # 200 packets per second
--pps=0.25    # 15 packets per minute
....

*-M, --mbps*=_string_::
  Replay packets at a given Mbps rate. Cannot be combined with *--multiplier*,
  *--pps*, *--oneatatime* or *--topspeed*.

*-t, --topspeed*::
  Replay packets as fast as possible. Cannot be combined with the other
  speed-control options.

*-o, --oneatatime*::
  Replay one packet at a time, waiting for user input between packets. Cannot be
  combined with the other speed-control options.

*--pps-multi*=_number_::
  Send this many packets for each timing interval, allowing accurate high rates.
  Requires *--pps*.

*--unique-ip*::
  Modify IP addresses each *--loop* iteration to generate unique flows, without
  altering packet CRC. Requires *--loop*.

*--unique-ip-loops*=_string_::
  Number of *--loop* iterations before a new unique IP is assigned (default 1).
  Requires *--unique-ip*.

*--netmap*::
  Write packets directly to a netmap-enabled network adapter, bypassing the
  driver for full line rate. Only available if compiled with netmap support.
  Can also be requested via a _netmap:<intf>_ or _vale:<intf>_ interface name.

*--nm-delay*=_number_::
  Seconds to delay after netmap is loaded, to ensure interfaces are up before
  transmit (default 10). Requires *--netmap*.

*--no-flow-stats*::
  Suppress the collection and printing of flow statistics.

*--flow-expiry*=_number_::
  Number of inactive seconds (per the pcap timestamps) before a flow is
  considered expired. Default 0 (no expiry); typical values are 30-120. Cannot
  be combined with *--no-flow-stats*.

*-P, --pid*::
  Print the PID of tcpreplay at startup.

*--stats*=_number_::
  Print statistics every X seconds, or every loop if 0. Timed delays are
  best-effort.

*-W, --suppress-warnings*::
  Suppress printing warning messages.

*--xdp*::
  Write packets directly to an AF_XDP-enabled network adapter, bypassing the
  network stack. Only available if compiled with AF_XDP (libxdp/libbpf) support.

*--xdp-batch-size*=_number_::
  Maximum number of packets submitted to the AF_XDP TX ring at once (1-4096,
  default 1). Only allowed at top speed; requires *--xdp* and *--topspeed*.
